# 로블록스 게임 추천 서비스 — 기술 설계 요약

> 지금까지 조사·실측으로 확정된 내용 정리 (팀 공유 + 작업 연속성용)

---

## 1. 서비스 개요

로블록스 유저에게 맞춤 게임을 추천하는 웹 서비스.
페이지 4개: ① 닉네임 입력 → ② 티어표 배치 → ③ 추천 게임 목록 → ④ 게임 상세(+영상)

---

## 2. 추천 알고리즘

### 입력
- **유저 즐겨찾기 게임** + **유저가 직접 추가한 게임** → 티어표(SSS/A/B/C)에 배치
- 티어별 가중치 부여 (상위 티어일수록 큼, SSS는 최대 2개만)

### 1단계 — 기본 점수 (연쇄 추천)
```
각 티어 게임
 → 그 게임을 플레이한 유저가 플레이한 게임 목록 (recommendations API)
 → 연쇄적으로 확장 (depth 1, 2, 3...)
 → depth에 따라 가중치 설정 (깊을수록 낮게)
 → 점수로 변환
 → 티어표 가중치를 곱함
```

### 2단계 — 팬 확장 (그룹 즐겨찾기 보강)
```
티어표 최상위 게임 2개만 선택
 → 그 게임 그룹 가입 + 즐겨찾기한 유저 여러 명 수집
 → 그 유저들의 즐겨찾기 게임 집계 (1명만 나온 게임은 제외 = 노이즈 컷)
 → 겹치는 순서(인원)에 따라 가중치 부여
 → 각 게임에 대해 다시 연쇄 추천 (1단계와 동일)
 → 최종 가중치는 유저가 직접 즐겨찾기/추가한 게임보다 낮게 보정
```

### 3단계 — 최종
- 1단계 + 2단계 점수 합산 → 점수 순으로 게임 나열

## 3. 사용할 API (전부 무인증 실측 검증됨, 2026-07-03)

### A. 입력/검색
| 용도 | API |
|---|---|
| 게임명 자동완성 | `apis.roblox.com/games-autocomplete/v1/get-suggestion/{입력}` |
| 게임명 검색 | `apis.roblox.com/search-api/omni-search?searchQuery={이름}` |
| 닉네임→유저ID | `POST users.roblox.com/v1/usernames/users` |

### B. 게임 정보/상세
| 용도 | API | 데이터 |
|---|---|---|
| 게임 상세 (배치 50개) | `games.roblox.com/v1/games?universeIds=..` | 이름·description·**장르(genre_l1/l2)**·visits·playing·favoritedCount·creator(소유그룹id) |
| 좋아요/싫어요 | `games.roblox.com/v1/games/votes?universeIds=..` | upVotes, downVotes |
| 게임 미디어 (스크린샷+**개발자 영상**) | `games.roblox.com/v2/games/{universeId}/media` | assetType(Image / **GamePreviewVideo**), imageId, videoHash |
| 게임 아이콘 | `thumbnails.roblox.com/v1/games/icons` | 아이콘 URL |
| 게임 스크린샷 | `thumbnails.roblox.com/v1/games/multiget/thumbnails` | 스크린샷 URL |
| placeId→universeId | `apis.roblox.com/universes/v1/places/{placeId}/universe` | universeId |

### C. 추천 (알고리즘 핵심)
| 용도 | API | 비고 |
|---|---|---|
| 연쇄 추천 (People Also Join) | `games.roblox.com/v1/games/recommendations/game/{universeId}` | **게임당 6개 상한**, 재귀로 확장 |

### D. 인기 게임 목록 (캐싱 대상 선정용)
| 용도 | API | 비고 |
|---|---|---|
| 인기/유입/수익 순위 | `explore-api/v1/get-sort-content?sortId={most-popular/top-playing-now/top-earning/top-revisited...}` | **차트당 95개 상한** |
| 장르별 인기 (17개 장르) | `sortId=trending-in-{장르}` | 장르별 큐레이션 |
| 차트 종류 목록 | `explore-api/v1/get-sorts` | 총 26종 (페이지네이션) |
| (보조) 넓은 게임 풀 | `api.rolimons.com/games/v1/gamelist` | 6,716개, 비공식, 브라우저 헤더 필요 |

### E. 그룹/팬 (알고리즘 2단계, 배치 전용)
| 용도 | API | 비고 |
|---|---|---|
| 게임 소유 그룹 | (B의 creator 필드) | |
| 그룹 멤버 목록 | `groups.roblox.com/v1/groups/{groupId}/users?cursor=..` | **커서 순차, 100명/호출, 동시성 불가** |

## 4. 핵심 제약 (실측 확인)

### Rate Limit / 속도
- **동시성**: 짧은 버스트는 150까지 OK, 200부터 커넥션 강제종료(10054). 1시간 지속은 동시성 4로 429 0%. 안전 운영값 50~100.
- **멤버 목록 조회**: limit 최대 100, 커서 순차라 **동시성 불가 → 못 빠르게 함** (100명당 0.65초 고정). 10,000명=65초, 100만명=1.8시간.
- **즐겨찾기 조회**: 유저별 독립이라 **동시성 가능** (빠르게 됨).

### 커서 특성
- 커서 = 그룹별 특정 멤버 위치의 암호화 값 + 해시 서명.
- **규칙 없음**: 그룹마다 다르고, N페이지 공통 커서 없음. 점프용 커서 생성 불가.
- **저장·재사용 가능**: 한 그룹의 특정 지점 커서를 저장하면 나중에 그 지점부터 재개 가능 (Desc는 신규가입으로 약간 밀림, Asc는 과거 고정이라 안정적).
- **활용**: 초인기 게임의 정착 유저 지점 커서를 배치로 미리 저장 → 재접근 빠르게. 단 그룹마다 개별, 주기적 갱신 필요.

### 팬 즐겨찾기 방식의 게임 의존성 (중요)
| 게임 유형 | 팬 취향 겹침 | 팬 즐겨찾기 수 | 이 방식 |
|---|---|---|---|
| 틈새/커뮤니티 (예: 한국 게임) | ✅ 뚜렷 | ✅ 40~50개 | **잘 됨** |
| 초대중/라이트 (예: RIVALS) | ❌ 제각각 | ❌ 대부분 1개 | **무용** |
- RIVALS 즐겨찾기자 = 뉴비/라이트 유저 많아 즐겨찾기 거의 안 함(평균 6.6개, 공통 게임 거의 0).
- 표본 크기: 5명=노이즈, **최소 15~20명** 필요.
- 즐겨찾기자 비율: Korea Army 2.5%(20명 확보에 ~800조회), RIVALS 31%(찾긴 쉬우나 결과 부실).

### 못 하는 것 (실측 확정)
- 특정 게임을 플레이/즐겨찾기한 유저 목록 (역방향 조회 불가)
- 유저 최근/오래 플레이한 게임, 플레이타임 (비공개)
- 그룹 가입 일자 (미공개, 가입 순서만 Asc/Desc)
- 계정 나이 대량 조회 (users API rate limit 빡빡, 14%만 성공)
- 홈 "Recommended For You" (`discovery-api/omni-recommendation`, 인증+본인 것만)

## 5. 두 추천 방식 비교 (실측)

| | recommendations (People Also Join) | 그룹 팬 즐겨찾기 (cofavorite) |
|---|---|---|
| 속도 | ⚡ 게임당 1호출 | 🐢 수백~수천 호출 |
| 데이터 | 로블록스 co-play 집계 | 우리가 모은 즐겨찾기 |
| 범용인기작 편중 | ✅ 자동 보정됨 | ❌ 오염됨 (RIVALS 상위) |
| 마이너 게임 | 너무 마이너한 것까지 | 표본만큼 |
| 커스텀 | ❌ 블랙박스 | ✅ 자유 튜닝 |
| 다양성(장르밖) | ❌ | ✅ |

**결론**: recommendations를 주력, 팬 즐겨찾기는 매니아 게임 한정 보강(배치 캐싱 전용).
