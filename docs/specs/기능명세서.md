## 필수 기능 상세

### F-01. 닉네임 입력

- **설명:** 사용자는 자신의 로블록스 닉네임을 입력하여 추천을 시작할 수 있다.
- **입력:** 로블록스 닉네임 (문자열)
- **처리:**
    - [ ]  입력한 닉네임을 로블록스 userId로 변환
    - [ ]  변환 성공 시 즐겨찾기 불러오기(F-02)로 진행
- **예외 상황:**
    - 존재하지 않는 닉네임 → "해당 닉네임의 유저를 찾을 수 없습니다" 안내
    - 빈 입력 → 입력 필드에 오류 표시
- **관련 화면:** 홈 / 시작 페이지 (`/`)
- **관련 API:** POST `users.roblox.com/v1/usernames/users` (외부)

### F-02. 즐겨찾기 불러오기

- **설명:** 사용자는 입력한 닉네임의 즐겨찾기 게임 목록을 화면에서 확인할 수 있다.
- **입력:** 변환된 userId (F-01 결과)
- **처리:**
    - [ ]  해당 유저의 즐겨찾기 게임 목록 조회
    - [ ]  각 게임의 이름·썸네일과 함께 목록 표시
- **예외 상황:**
    - 즐겨찾기 비공개/0개 → "표시할 즐겨찾기가 없습니다. 인기 게임을 둘러보세요" 안내 (F-10로 유도)
- **관련 화면:** 티어 배치 페이지 (`/tier`)
- **관련 API:** GET `games.roblox.com/v2/users/{userId}/favorite/games` (외부)

### F-03. 티어 배치

- **설명:** 사용자는 불러온 즐겨찾기 게임들을 S/A/B/C/D 티어표에 드래그하여 선호도를 표현할 수 있다.
- **입력:** 게임 카드 드래그앤드롭 (게임 → 티어)
- **처리:**
    - [ ]  S/A/B/C/D 5단계 티어표 제공
    - [ ]  게임 카드를 원하는 티어로 드래그하여 배치
    - [ ]  배치하지 않은 게임은 "미분류"로 처리 (추천 시 제외 또는 낮은 가중치)
    - [ ]  티어별 가중치 부여 (S=5, A=4, B=3, C=2, D=1)
- **예외 상황:**
    - 아무 게임도 배치하지 않고 추천 요청 → "게임을 하나 이상 배치해주세요" 안내
- **관련 화면:** 티어 배치 페이지 (`/tier`)
- **관련 API:** (없음 — 클라이언트 상태)

### F-04. 추천 생성

- **설명:** 사용자는 티어 배치를 바탕으로 맞춤 추천 게임 목록을 생성할 수 있다.
- **입력:** 티어 배치 결과 (게임별 티어)
- **처리:**
    - [ ]  각 티어 게임의 연관 게임(cofavorite / 유사도)을 조회
    - [ ]  후보별 점수 = Σ(티어 가중치 × 연관 강도 × 신뢰도)
    - [ ]  유명도 보정 적용 (범용 인기작 편중 방지)
    - [ ]  이미 즐겨찾기한 게임 제외
    - [ ]  상위 N개 추천 결과 반환
    - [ ]  ※ 계산 로직 상세는 별도 "추천 알고리즘 설계 문서" 참고
- **예외 상황:**
    - 연관 데이터 조회 실패 → 캐시/폴백 데이터로 대체, 실패 시 "추천을 생성하지 못했습니다" 안내
- **관련 화면:** 로딩 → 추천 결과 페이지 (`/result`)
- **관련 API:** (내부) GET `/api/recommend`

### F-05. 추천 결과 카드 (장르별 구분)

- **설명:** 사용자는 추천된 게임들을 썸네일·이름 카드로 확인하고, 장르별로 구분해서 볼 수 있다.
- **입력:** (없음 — F-04 결과 표시)
- **처리:**
    - [ ]  추천 게임을 썸네일 + 이름 카드로 그리드 표시
    - [ ]  장르(genre) 기준으로 그룹핑 또는 필터 제공
    - [ ]  카드 클릭 시 게임 상세(F-06)로 이동
- **예외 상황:**
    - 추천 결과 0개 → "조건에 맞는 게임을 찾지 못했습니다" 안내
- **관련 화면:** 추천 결과 페이지 (`/result`)
- **관련 API:** (내부) GET `/api/recommend`, 썸네일: `thumbnails.roblox.com` (외부)

### F-06. 게임 상세 보기

- **설명:** 사용자는 추천 카드를 눌러 해당 게임의 상세 정보를 확인할 수 있다.
- **입력:** 게임 선택 (universeId)
- **처리:**
    - [ ]  게임명·설명·장르·현재 동접·방문수·좋아요 표시
    - [ ]  아이콘/스크린샷 표시
    - [ ]  로블록스 게임 페이지 바로가기 링크 제공
    - [ ]  (선택) 우측(또는 하단)에 관련 유튜브 영상 패널 노출 (F-10)
- **예외 상황:**
    - 게임 정보 조회 실패 → "게임 정보를 불러올 수 없습니다" 안내
- **관련 화면:** 게임 상세 페이지/모달 (`/game/:id`)
- **관련 API:** GET `games.roblox.com/v1/games`, `thumbnails.roblox.com` (외부), (내부) GET `/api/game/:id`

## 선택 기능 상세

### F-10. 게임 관련 유튜브 영상 (쇼츠형)

- **설명:** 사용자는 게임 상세 화면에서 해당 게임을 플레이하는 유튜브 영상을, 쇼츠처럼 위아래로 넘기며 쉽게 감상할 수 있다.
- **입력:** 게임명 (검색 키워드)
- **처리:**
    - [ ]  게임명으로 유튜브 관련 영상 검색
    - [ ]  검색 결과를 세로 스와이프/넘김 UI(쇼츠형)로 순차 재생
    - [ ]  각 영상은 유튜브 임베드 플레이어로 재생
    - [ ]  ※ API 할당량 절약을 위해 게임별 검색 결과를 DB에 캐싱하여 재사용
- **예외 상황:**
    - 관련 영상 없음 → "관련 영상을 찾지 못했습니다" 안내
    - API 할당량 초과 → 캐시된 결과로 대체
- **구현 조건:** 필수 기능 완료 후 잔여 기간에 따라 착수 여부 결정
- **관련 화면:** 게임 상세 페이지 영상 패널
- **관련 API:** YouTube Data API v3 `search.list` (외부, 캐싱 필수)

> **⚠️ F-10 제약 사항 (구현 시 인지 필요)**
> - YouTube `search.list`는 호출당 100유닛, 무료 하루 한도 10,000유닛 → 하루 검색 약 100회 제한. **게임별 검색 결과 캐싱이 필수.**
> - "유튜브 쇼츠"만 필터링하는 것은 API가 잘 지원하지 않음 → **일반 영상을 쇼츠처럼 넘기는 UI**로 구현.
> - 게임명 검색이라 무관한 영상이 일부 섞일 수 있음 (검색 품질 한계).

### F-11. 인기 게임 탐색

- **설명:** 사용자는 닉네임 입력 없이도 현재 인기/트렌드 게임을 둘러볼 수 있다.
- **구현 조건:** 필수 기능 완료 후 잔여 기간에 따라 착수 여부 결정
- **관련 API:** `apis.roblox.com/explore-api` (차트, 외부)

### F-12. 추천 결과 공유

- **설명:** 사용자는 자신의 추천 결과를 이미지 또는 링크로 공유할 수 있다.
- **구현 조건:** 필수 기능 완료 후 잔여 기간에 따라 착수 여부 결정
