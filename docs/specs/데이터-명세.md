# 사용할 데이터 명세 (저장 대상)

> 각 API 응답 필드 중 **무엇을 저장할지** 정리. API 상세는 `api-명세.md` 참고.
> DB는 "느린 API를 피하기 위한 캐시 계층". 실시간 조회 최소화가 목적.

---

## 1. 게임 메타데이터 (games 테이블)

출처: B-1(`/v1/games`) + B-2(votes) + C-1(recommendations도 일부 제공)

| 저장 필드 | 출처 필드 | 저장? | 용도 |
|---|---|---|---|
| universe_id | id | ✅ PK | 식별 |
| place_id | rootPlaceId | ✅ | URL 링크 |
| name | name | ✅ | 표시 |
| description | description / gameDescription | ✅ | 상세 표시 |
| genre_l1 | genre_l1 | ✅ | 장르 필터/탐색 |
| genre_l2 | genre_l2 | ✅ | 서브장르 |
| playing | playing / playerCount | ✅ | 유명도 보정, 표시 |
| visits | visits | ✅ | 유명도 보정 |
| favorited_count | favoritedCount | ✅ | 유명도 |
| up_votes | upVotes / totalUpVotes | ✅ | 평점 |
| down_votes | downVotes / totalDownVotes | ✅ | 평점 |
| creator_group_id | creator.id (type=Group일 때) | ✅ | 2단계 그룹 접근 |
| minimum_age | minimumAge | ✅ | 연령 필터 |
| icon_url | (F-1 썸네일) | ✅ | 카드 표시 |
| fan_cacheable | (판별해서 기록) | ✅ | 팬 방식 적합 게임 플래그 |
| updated_at | (수집 시각) | ✅ | 캐시 신선도 |
| maxPlayers, created, updated, price | | ❌ | 미사용 |

## 2. 게임 미디어 (game_media 테이블)

출처: B-3(`/v2/games/{id}/media`) + F-2(썸네일 URL 변환)

| 저장 필드 | 출처 | 저장? | 용도 |
|---|---|---|---|
| universe_id | (요청 게임) | ✅ | 연결 |
| asset_type | assetType | ✅ | Image / GamePreviewVideo 구분 |
| media_url | imageId→F-2 변환 URL | ✅ | 스크린샷 표시 |
| youtube_video_id | videoId | ✅ | **개발자 영상(유튜브ID)** 재생 |
| sort_order | (배열 순서) | ✅ | 표시 순서 |

## 3. 연쇄 추천 (game_recommendations 테이블)

출처: C-1(`recommendations`)

| 저장 필드 | 출처 | 저장? | 용도 |
|---|---|---|---|
| from_universe_id | (요청 게임) | ✅ PK | 시드 |
| to_universe_id | universeId | ✅ PK | 연관 게임 |
| rank | (배열 순위 1~6) | ✅ | 연관 강도 |
| fetched_at | (수집 시각) | ✅ | 캐시 신선도 |
- 참고: recommendations가 연관 게임의 상세(장르·동접 등)도 주므로, 그걸로 games 테이블도 함께 채움 (조회 절약)

## 4. 팬 데이터 (2단계, 배치 전용)

### 4-1. 팬 유저 목록 (game_fans)
출처: E-2(그룹멤버) + E-3(즐겨찾기)로 판별

| 저장 필드 | 저장? | 용도 |
|---|---|---|
| seed_universe_id | ✅ PK | 어느 게임 팬인가 |
| user_id | ✅ PK | 팬 유저 |
| collected_at | ✅ | 수집 시각 |

### 4-2. 팬 즐겨찾기 원시 (fan_favorites)
출처: E-3. **가중치 재튜닝 위해 원시 보관**

| 저장 필드 | 출처 | 저장? |
|---|---|---|
| user_id | (팬) | ✅ PK |
| fav_universe_id | id | ✅ PK |

### 4-3. 팬 공통 즐겨찾기 집계 (game_cofavorite)
출처: 4-2를 집계한 결과 (배치 계산)

| 저장 필드 | 저장? | 용도 |
|---|---|---|
| seed_universe_id | ✅ PK | 기준 게임 |
| related_universe_id | ✅ PK | 함께 즐겨찾기된 게임 |
| overlap_count | ✅ | 겹친 팬 수 (가중치) |
| sample_size | ✅ | 표본 크기 (신뢰도) |
| computed_at | ✅ | 계산 시각 |

## 저장 안 하는 데이터 (실시간만, 캐싱 불필요)

| 데이터 | 이유 |
|---|---|
| 닉네임→ID 변환 | 매번 실시간 (가벼움) |
| 유저의 즐겨찾기 목록 (추천 요청 시) | 실시간 (그 순간의 취향) |
| 티어 배치 상태 | 클라이언트 상태 (로그인 없음) |
| 자동완성 결과 | 실시간 |

## 핵심 요약

- **games** = 모든 게임 캐시 (표시 + 유명도 보정)
- **game_recommendations** = 연쇄 추천 캐시 (알고리즘 1·2단계)
- **game_cofavorite** + **game_fans** + **fan_favorites** = 2단계 팬 데이터 (배치 필수)
- **group_cursors** = 정착 유저 재접근 (커서 저장)
- **chart_snapshot** = 캐싱 대상 선정
- **collect_queue** = lazy population
- 유저 계정/로그인 테이블 **없음**
